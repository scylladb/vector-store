# Dataset Preparation

This document covers preparing data for the FTS benchmark. For benchmark
execution see [README.md](README.md).

## TSV Format

All data files are tab-separated, no header quoting, no escape characters.

### `documents.tsv`

Corpus. Columns: `doc_id`, `body`.

```
doc_00001	Tetraodontiformes is an order of fish...
doc_00002	Climate change refers to long-term shifts...
```

### `queries_<set>.tsv`

Query sets for the search phase. Columns: `query_id`, `text`.

```
0	How do fish adapt to climate change?
2	What are the side effects of ibuprofen?
```

The `<set>` name determines which query set the orchestrator runs. Standard
sets from BEIR are stored in `queries_natural.tsv`. Synthetic query sets
(`term_rare`, `phrase`, `boolean_and`, etc.) are generated by `--synthetic-queries`.

### `qrels_natural.tsv`

Relevance judgments for accuracy metrics (recall, precision, MRR, nDCG).
Columns: `query_id`, `doc_id`, `relevance`.

```
0	doc_00001	1
0	doc_00005	1
2	doc_00003	1
```

Relevance is `1` (relevant) — BEIR datasets use binary relevance. Only the
`natural` query set has matching qrels, so accuracy is only computed for
natural queries.

## Source: BEIR

### Basic usage

```bash
python prepare_dataset.py \
    --source beir --dataset scifact \
    --out /tmp/scifact
```

This downloads the `scifact` corpus, queries, and qrels from HuggingFace and
writes them as TSV files.

### How loading works

`prepare_dataset.py` first looks for Parquet cache files under
`BeIR/{dataset}/{config}/`. Parquet is always used when available — it is
faster and avoids the legacy HF loading script. The Parquet path loads
**all data** with `split="train"` (the BEIR convention stores everything in
a single `train` shard).

If no Parquet files exist, it falls back to HuggingFace `load_dataset()` with
the requested split name.

### Split control

By default (`--split` unset), queries and qrels from **all splits are merged**
into a single dataset. This is useful for getting the largest possible corpus
for throughput testing.

```bash
# All data merged (default)
python prepare_dataset.py --source beir --dataset scifact --out /tmp/scifact
# → 5183 docs, 1109 queries, 1258 qrels

# Test split only
python prepare_dataset.py --source beir --dataset scifact --split test --out /tmp/scifact-test
# → 5183 docs, 300 queries, 339 qrels
```

When `--split` is set:
- The corpus is the same regardless of split (BEIR stores all docs in one
  collection).
- Queries are loaded from Parquet (all data), then **filtered** to only those
  with qrels entries in the requested split's qrels file. This ensures every
  query in the output has a relevance judgment.

When `--split` is unset:
- Queries are loaded from Parquet (all data).
- Qrels are loaded from the `qrels` config in the main `BeIR/{dataset}` repo
  (Parquet preferred). If that config is not found, the script falls back to
  downloading from `BeIR/{dataset}-qrels` — all `.tsv` files merged with dedup.
- Queries are filtered to only those with matching qrels entries (same as the
  `--split` path). With all splits merged, most queries will have a match, but
  the filter still runs.

## Source: Custom TSV

```bash
python prepare_dataset.py \
    --source custom \
    --corpus my_docs.tsv \
    --queries my_queries.tsv \
    --qrels my_qrels.tsv \
    --out /tmp/mydata
```

Each TSV must follow the format described above. The `--queries` and `--qrels`
flags are optional — the benchmark runs without accuracy metrics if no qrels
are provided, and runs latency-only if no queries are provided.

## Source: Prepared (reprocessing)

Use an existing prepared directory as input. This is useful for applying
scaling or sharding to a previously prepared dataset without re-downloading.

```bash
# First download
python prepare_dataset.py --source beir --dataset scifact --out /tmp/scifact

# Then re-process with scaling
python prepare_dataset.py \
    --source prepared --prepared-dir /tmp/scifact \
    --scale 5 --out /tmp/scifact-5x
```

## Scaling

### Replicate mode (`--scale-mode replicate`, default)

Each document is copied N times. The first copy keeps the original `doc_id`,
subsequent copies get `{doc_id}_copy_1`, `{doc_id}_copy_2`, etc.

Qrels are **mirrored** to all copies — a qrel entry `(q1, doc_001, 1)` becomes
`(q1, doc_001, 1)`, `(q1, doc_001_copy_1, 1)`, `(q1, doc_001_copy_2, 1)`.
This means accuracy metrics still work correctly since every copy has the same
body text and counts as relevant.

```bash
# 5183 docs → 51,830 docs (accuracy preserved)
python prepare_dataset.py --source beir --dataset scifact \
    --scale 10 --out /tmp/scifact-10x
```

### Synthetic mode (`--scale-mode synthetic`)

Generates entirely new documents by shuffling fragments from the original
corpus. Originals are **discarded** — the output is purely synthetic.

**Requirement:** Synthetic mode requires either `--target-docs` or `--scale > 1`.
Running with default `--scale 1` and no `--target-docs` will error.

**Fragment units** (controlled by `--fragment-unit`):

| Unit | Split rule | Best for |
|------|-----------|----------|
| `sentence` | Regex on `[.!?]` whitespace | Medium docs (50–500 tok) |
| `paragraph` | Split on `\n\n` | Long docs (>500 tok) |
| `word` | Whitespace | Very short docs (<50 tok) |
| `auto` (default) | Heuristic from avg doc length | — |

**Auto-detect heuristic:**

| Mean doc tokens | Unit chosen |
|----------------|-------------|
| >500 | `paragraph` |
| 50–500 | `sentence` |
| <50 | `word` |

**Document assembly** works by repeatedly sampling random fragments from the
pool until the accumulated token count reaches `target_doc_size`
(corpus mean if not specified, within ±20% tolerance).

```bash
# 10,000 synthetic docs, sentences, ~200 tokens each
python prepare_dataset.py --source beir --dataset scifact \
    --scale-mode synthetic --target-docs 10000 \
    --target-doc-size 200 --out /tmp/scifact-synth

# 50,000 synthetic docs, paragraphs, ~500 tokens each
python prepare_dataset.py --source beir --dataset trec-covid \
    --scale-mode synthetic --target-docs 50000 \
    --fragment-unit paragraph --target-doc-size 500 \
    --out /tmp/trec-covid-synth
```

**Qrels**: emptied in synthetic mode — no relevance judgments match the
generated document IDs. The benchmark automatically skips accuracy metrics.

**Queries**: preserved. Natural queries from the source are kept as-is for
latency benchmarking. Synthetic queries can still be generated via
`--synthetic-queries`.

## Synthetic Queries

Generated on top of any source, controlled by `--synthetic-queries N` (default
0, disabled). Uses token frequency analysis of the corpus to bucket terms.

| File | Type | Example | Description |
|------|------|---------|-------------|
| `queries_term_rare.tsv` | Rare single terms | `tetraodontiformes` | From bottom ⅓ of term freq |
| `queries_term_medium.tsv` | Medium-frequency terms | `climate` | Middle ⅓ |
| `queries_term_common.tsv` | Common terms | `study` | Top ⅓ |
| `queries_phrase.tsv` | 2–4 token phrases | `global temperature rise` | Random n-grams from docs |
| `queries_boolean_and.tsv` | Term1 AND Term2 | `climate AND policy` | Random term pair |
| `queries_boolean_or.tsv` | Term1 OR Term2 | `solar OR wind` | Random term pair |
| `queries_boolean_not.tsv` | Term1 NOT Term2 | `fossil NOT coal` | Random term pair |

Synthetic queries have **no qrels** — they are useful for latency and
throughput measurement, not accuracy.

## Sharding

Split documents into N shard files for multi-process loading:

```bash
python prepare_dataset.py --source beir --dataset scifact \
    --shards 8 --out /tmp/scifact-sharded
```

Output: `shards/documents_000.tsv` through `documents_007.tsv`, each with
approximately equal row count.

## CLI Reference

```
usage: prepare_dataset.py [-h] --source {beir,beir-local,custom,prepared}
                          [--dataset DATASET] [--split SPLIT]
                          [--beir-dir BEIR_DIR] [--corpus CORPUS]
                          [--queries QUERIES] [--qrels QRELS]
                          [--prepared-dir PREPARED_DIR] --out OUT
                          [--scale SCALE] [--scale-mode {replicate,synthetic}]
                          [--fragment-unit {auto,sentence,paragraph,word}]
                          [--target-doc-size TARGET_DOC_SIZE]
                          [--target-docs TARGET_DOCS]
                          [--max-docs MAX_DOCS] [--max-queries MAX_QUERIES]
                          [--shards SHARDS]
                          [--synthetic-queries SYNTHETIC_QUERIES]
                          [--boolean-and-pool {rare,medium,common,medium+common,rare+medium}]
                          [--boolean-or-pool {rare,medium,common,medium+common,rare+medium}]
                          [--boolean-not-pool {rare,medium,common,medium+common,rare+medium}]
                          [--seed SEED]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--source` | (required) | `beir`, `beir-local`, `custom`, or `prepared` |
| `--dataset` | — | BEIR dataset name (for `beir`) |
| `--split` | `None` (all data) | BEIR dataset split (`test`, `train`, `dev`, ...) |
| `--beir-dir` | — | Local BEIR directory (for `beir-local`) |
| `--corpus` | — | Custom corpus TSV (for `custom`) |
| `--queries` | — | Custom queries TSV (for `custom`) |
| `--qrels` | — | Custom qrels TSV (for `custom`) |
| `--prepared-dir` | — | Existing prepared dir (for `prepared`) |
| `--out` | (required) | Output directory |
| `--scale` | `1` | Document scale factor (no scaling at 1) |
| `--scale-mode` | `replicate` | `replicate` (copy + qrel mirror) or `synthetic` (fragment shuffle) |
| `--fragment-unit` | `auto` | Fragment granularity for synthetic mode |
| `--target-doc-size` | corpus mean | Target tokens per synthetic document |
| `--target-docs` | — | Exact synthetic doc count (instead of `--scale`) |
| `--max-docs` | — | Cap original documents before scaling |
| `--max-queries` | — | Cap natural queries (BEIR only) |
| `--shards` | `1` | Split into N shard files |
| `--synthetic-queries` | `0` | Synthetic queries per bucket (0 to disable) |
| `--boolean-and-pool` | `medium+common` | Term pool for AND boolean queries (`rare`, `medium`, `common`, `medium+common`, `rare+medium`) |
| `--boolean-or-pool` | `rare` | Term pool for OR boolean queries |
| `--boolean-not-pool` | `medium+common` | Term pool for NOT boolean queries |
| `--seed` | `42` | Random seed |

## Manifest

Every prepared dataset contains a `manifest.json`:

| Field | Type | Description |
|-------|------|-------------|
| `source` | string | Data source (`beir`, `beir-local`, `custom`, `prepared`) |
| `dataset` | string | BEIR dataset name (for `beir` source) |
| `split` | string/null | BEIR split used (`"all"` for `beir` with no `--split`; `null` for other sources without `--split`) |
| `beir_dir` | string | Local BEIR directory (for `beir-local` source) |
| `corpus` | string | Custom corpus TSV path (for `custom` source) |
| `prepared_dir` | string | Prepared directory path (for `prepared` source) |
| `documents` | int | Total document count (after scaling) |
| `original_doc_count` | int | Document count before scaling |
| `scale_mode` | string | `replicate` or `synthetic` |
| `scale_factor` | int/float | Applied scale factor |
| `fragment_unit` | string | Fragment unit used (synthetic mode only) |
| `has_qrels` | bool | Whether qrels were written |
| `synthetic_queries` | bool | Whether synthetic queries were generated |
| `synthetic_query_count_per_bucket` | int | Count per bucket |
| `num_shards` | int | Number of shard files |
| `queries_natural` | int | Natural query count |
| `qrels_natural` | int | Qrels entry count |
| `qrels_stats` | object | Statistics: total entries, queries with qrels, relevant/query dist, grade dist |
| `queries_<set>` | int | Synthetic query count per set |
| `doc_token_stats` | object | Token stats (see below) |
| `queries_<set>_token_stats` | object | Per-query-set token statistics |

`doc_token_stats` differs by mode:
- **Non-synthetic:** `{total_token_count, mean_tokens_per_doc, p50_tokens_per_doc, p90_tokens_per_doc, max_tokens_per_doc}`
- **Synthetic (sampled):** `{total_token_count: null, sample_size, total_docs_seen, mean_tokens_per_doc, p50_tokens_per_doc, p90_tokens_per_doc, max_tokens_per_doc}`
